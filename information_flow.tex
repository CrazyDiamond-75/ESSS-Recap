\chapter{Information Flow}
This is the second part of the lecture, where we will define and discuss different security properties on networks.

One example where security was violated by an information flow mess up was the \emph{Meltdown} attack, where CPU pipelining lead to a process gaining access to forbidden memory.
In this example, the security policy is isolation between (unrelated) processes.
Processes may communicate with the kernel, but not directly with each other, this was violated by the attack.
